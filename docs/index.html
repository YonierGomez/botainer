01 — Características

Todo lo que necesitas
para gestionar Docker.

Desde monitoreo en tiempo real hasta creación de contenedores paso a paso.

📊

Monitoreo en tiempo real

CPU, RAM y healthchecks de cada contenedor con actualización instantánea.

🔔

Notificaciones automáticas

Alertas de inicio, caída, reinicio y pausa sin configuración extra.

📁

Docker Compose completo

Up, Down, Restart, Pull y PS por proyecto con detección automática.

🛠️

Crear contenedores

Asistente paso a paso para Docker Run y Compose. Genera el comando o YAML listo para usar.

📜

Logs avanzados

Resaltado de errores, filtros, refresh y descarga como archivo .log.

🔧

Diagnóstico automático

Detecta contenedores caídos, unhealthy, alto CPU e imágenes sin usar.

🔍

Búsqueda universal

Busca en contenedores, imágenes y volúmenes con un solo comando.

⭐

Favoritos

Acceso rápido a los contenedores que más usas, persistente por usuario.

🔐

Seguridad por whitelist

Solo los usuarios autorizados pueden interactuar con el bot.

03 — Instalación

En marcha
en 3 pasos.

Solo necesitas Docker, Docker Compose y un token de Telegram.

01

Clonar el repositorio

Descarga el proyecto en tu servidor Linux.

02

Configurar el token

Copia el archivo de ejemplo y añade tu token de @BotFather.

03

Iniciar el bot

Docker construye la imagen y arranca el contenedor automáticamente.

# 1. Clonar git clone https://github.com/YonierGomez/botainer.git cd botainer # 2. Configurar token cp .env.example .env # Edita .env → TELEGRAM_BOT_TOKEN=tu_token # 3. Iniciar docker compose up -d --build docker logs -f botainer # → Bot iniciado: @tu_bot
04 — CI/CD

Build multi-arch
automatizado.

Pipeline semanal que detecta nuevas versiones de Alpine y publica imágenes para amd64 y arm64 sin QEMU.

Docker Multi-Arch CI

El workflow se ejecuta cada lunes a las 08:00 UTC. Compara la versión de Alpine con el último release y, si hay cambios, construye en runners nativos, fusiona el manifiesto multi-arch y crea un GitHub Release con changelog.
